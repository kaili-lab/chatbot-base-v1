# Task 05: 首页落地页

## 依赖

- 前置 task：Task 01
- 相关文件：`src/app/page.tsx`

## 需求描述

实现产品落地页，作为未登录用户的入口页面。已登录用户访问首页时重定向到 `/chat`。这是一个纯静态展示页面，不涉及业务逻辑。

## UI 参考

- 参考截图：用户在 AI Studio 中生成的落地页截图（待用户提供）
- 落地页独立于 (app) 布局，不使用侧边栏

## 实现要点

### 页面结构

- **顶部导航**：Logo + 右侧"登录"、"注册"按钮（链接到 /login、/register，页面暂时用占位）
- **Hero 区域**（居中）：
  - 大标题：产品名称和一句话价值主张
  - 副标题：简短描述产品能力
  - CTA 按钮："开始使用"（链接到 /register）
- **特性展示**：2-3 个特性卡片（上传文档、语义检索、AI 问答），使用 shadcn/ui Card 组件
- **页脚**：简单的版权信息

### 登录状态判断

- 使用 Server Component 检查用户登录状态
- 已登录 → `redirect('/chat')`
- 未登录 → 展示落地页

### 样式

- 支持亮色/暗色主题
- 响应式布局（移动端适配）
- 保持简洁，留白充足

## 验证标准

### 预期行为

- 未登录状态访问 `/` 看到落地页
- 页面包含导航栏、Hero 区域、特性卡片、页脚
- "登录"、"注册"按钮链接正确（页面暂时 404 没关系，路由在 Task 06 实现）
- 亮色/暗色主题下都能正常显示
- 移动端布局正常
- `pnpm build` 构建成功
